## 2. Architettura & Librerie (ARES ERP MASTER)
- **Tipo di Progetto:** Applicazione Web HMI Avanzata (Singola Pagina HTML, CSS integrato, JS vanilla).
- **Librerie Principali:** Leaflet.js (GIS Cartografia), Chart.js (Grafici BI), Supabase (BaaS), HLS.js (Flussi video Telecamere), Force-Graph (Nodi per Fascicoli).
- **Stile Visivo:** Dark Mode profonda (`#0c0d12`), effetti Glassmorphism, design liquido ultra-responsivo. Layout scalabile.
- **Struttura Sidebar (Holographic Glass):** L'HMI è divisa in due barre fluttuanti indipendenti (glassmorphism):
  - **Left Sidebar (Azzurra):** Navigazione Moduli (CRM, Planning, ecc.).
  - **Right Sidebar (Verde Smeraldo):** Controlli di Sistema, Zoom, Database Cloud, Presenze Online e GPS.
  - Entrambe hanno linguette laterali "Neon Tab" a scomparsa automatica con sistema di "Pin" per fissarle a schermo.

## 3. Moduli HMI (Sezioni dell'Applicazione)
Il routing interno avviene nascondendo le viste (class `.view-section`) tramite la funzione JS `show(id)`:
1. **dashboard:** (`refreshDashboard()`) - Homepage con alert e riepiloghi.
2. **operativa:** (`initGisMap()`) - Sala Operativa GIS (Mappa interattiva, radar meteo PiP).
3. **grafana:** (`updateAll()`) - BI Prestigioso con grid KPI e Chart.js.
4. **planning:** (`renderAllCalendars()`) - Calendario e pianificazione task.
5. **risorse:** (`renderResources()`) - Resource Manager (Personale, Mezzi, ecc).
6. **preventivatore:** (`renderPreventivi()`) - Sistema di preventivazione.
7. **videomanager:** (`renderCameraManager()`) - Gestione flussi video (IP/HLS).
8. **utenti:** (`renderUsers()`) - Amministrazione e accessi.
9. **crm:** (`renderCRM()`) - Gestione rubrica clienti.
10. **fascicoli:** (`renderFascicoli()`) - Gestione dossier tecnici. Include un sistema 3D/2D "Force-Graph" integrato per esplorare connessioni semantiche tra Fascicoli, Preventivi e CRM.

## 4. Struttura Dati (Supabase Cloud Sync Engine)
Il database PostgreSQL è gestito tramite `@supabase/supabase-js@2`. 
Tutte le tabelle principali che vengono sincronizzate / lette tramite client sono precedute dal prefisso `ares_`:
- `ares_users` (Utenti e permessi HMI)
- `ares_presence` (Presenze e status online)
- `ares_planning` (Appuntamenti e task calendario)
- `ares_crm` (Clienti, contatti, log CRM)
- `ares_resources` (Gestione risorse e flotta)
- `ares_preventivi` (Storico preventivazioni)
- `ares_cameras` (URL streaming e telecamere associate)
- `ares_fascicoli` (Dossier e associazioni a commesse)

## 5. Workflow Raccomandato
1. Analizza la richiesta dell'utente.
2. Controlla e naviga in `index.html` sfruttando la struttura a sezioni conosciuta.
3. Se l'utente chiede modifiche ai dati, interfaccia la query sulle tabelle di Supabase qui descritte.
4. Usa l'eseguibile `git.exe` in `C:\Program Files\Git\cmd\` per pushare autonomamente, senza chiedere permessi.

## 6. Architettura HMI Multi-Schermo
Le view possono essere sganciate in finestre popup separate aggiungendo il parametro '?detached=true&view=nomeview' all'URL. In questa modalità, l'app nasconde la sidebar e l'header. La sincronizzazione dei dati tra la finestra principale e quelle sganciate è gestita tramite un costrutto BroadcastChannel ('ares_channel'). Le funzioni per comunicare sono: `window.sendAresBroadcast(action, payload)`.

## 7. Funzionalità Locali Avanzate
- **Radar Multimedia (File System Access API):** All'interno della Pagina 2 del Fascicolo, l'operatore può usare una drop-zone (Radar) per salvare file fisici senza passare per il cloud. Il browser richiede il permesso alla cartella e ne memorizza l'handle tramite IndexedDB (`AresRadarDB`).
- **Geocoding Integrato & Autocomplete:** Gli indirizzi dei Fascicoli a Pagina 1 e dei Preventivi vengono convertiti automaticamente in coordinate `lat` e `lng` interrogando le API pubbliche di Nominatim (OpenStreetMap). Durante la digitazione è presente un autocomplete live (dropdown suggerimenti) basato sui dati Nominatim.
- **Mappa "Attività e Cantieri":** Oltre a traffico e meteo, la mappa GIS globale dispone di un layer generato in automatico che posiziona i cantieri attivi; un clic sul cantiere apre direttamente la UI olografica del Fascicolo pertinente. I marker in mappa supportano tooltip custom (Neon Glassmorphism).
- **Pianificazione (Modal e Task):** I task e le scadenze del fascicolo vengono gestiti tramite finestre modali avanzate in stile glassmorphism con data/time picker nativo, integrando estetica futuristica.
