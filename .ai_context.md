## 2. Architettura & Librerie (ARES ERP MASTER)
- **Tipo di Progetto:** Applicazione Web HMI Avanzata (Singola Pagina HTML, CSS integrato, JS vanilla).
- **Librerie Principali:** Leaflet.js (GIS Cartografia), Chart.js (Grafici BI), Supabase (BaaS), HLS.js (Flussi video Telecamere), html2canvas.
- **Stile Visivo:** Dark Mode profonda (`#0c0d12`), effetti glassmorphism (sfocature), design liquido ultra-responsivo. Layout scalabile con Zoom e Media Queries.

## 3. Moduli HMI (Sezioni dell'Applicazione)
Il routing interno avviene nascondendo le viste (class `.view-section`) tramite la funzione JS `show(id)`:
1. **dashboard:** (`refreshDashboard()`) - Homepage con alert e riepiloghi.
2. **operativa:** (`initGisMap()`) - Sala Operativa GIS (Mappa interattiva, radar meteo PiP).
3. **grafana:** (`updateAll()`) - BI Prestigioso con grid KPI e Chart.js.
4. **planning:** (`renderAllCalendars()`) - Calendario e pianificazione task.
5. **risorse:** (`renderResources()`) - Resource Manager (Personale, Mezzi, ecc).
6. **preventivatore:** (`renderPreventivi()`) - Sistema di preventivazione.
7. **videomanager:** (`renderCameraManager()`) - Gestione flussi video (IP/HLS).
8. **utenti:** (`renderUsers()`) - Amministrazione e accessi.

## 5. Workflow Raccomandato
1. Analizza la richiesta dell'utente.
2. Controlla e naviga in `index.html` sfruttando la struttura a sezioni conosciuta.
3. Se l'utente chiede modifiche ai dati, interfaccia la query sulle tabelle di Supabase qui descritte.
4. Usa l'eseguibile `git.exe` in `C:\Program Files\Git\cmd\` per pushare.

## 6. Architettura HMI Multi-Schermo
Le view possono essere sganciate in finestre popup separate aggiungendo il parametro '?detached=true&view=nomeview' all'URL. In questa modalità, l'app nasconde la sidebar e l'header. La sincronizzazione dei dati tra la finestra principale e quelle sganciate è gestita tramite un costrutto BroadcastChannel ('ares_channel'). Le funzioni per comunicare sono: window.sendAresBroadcast(action, payload).
